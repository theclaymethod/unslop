The migration project began on a rainy Saturday. I wanted to move the old database to new hardware over one weekend.

Moreover, the schema felt stable from the start, offering a solid base for the cutover, reinforcing my early confidence.

Furthermore, the export ran for two hours, streaming rows to disk, revealing a few corrupt records.

However, the import needed three passes to finish, retrying slowly between failures, testing whatever patience I had left.

In addition, one rollback script saved the whole cutover, catching the last error, stabilizing the migration at last.

Ultimately, the migration project proved that one weekend and $40 of cloud credit can move a tired old database.
